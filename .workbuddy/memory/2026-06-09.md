# 2026-06-09 工作日志

## 梳理 LabNote 项目代码逻辑与功能

完成了对 labnote 项目的完整代码分析，项目为 **Electron + React + TypeScript** 桌面应用，用于化学实验记录管理。

### 技术栈
- Electron 28 + better-sqlite3（主进程）
- React 18 + React Router 6 + Tailwind CSS（渲染进程）
- Drizzle ORM schema（定义用，实际查询通过 IPC 调用）
- 数据库文件存放于用户自选的 dataPath（默认 ~/Documents/LabNoteData/labnote.db）

### 核心模块
1. **实验记录**（核心）：记录反应物/催化剂/溶剂/条件/产率，支持图片附件
2. **课题管理**：项目归属、进度追踪、创新点/任务记录
3. **模板库**：实验条件模板保存与复用，统计使用次数
4. **试剂库**：试剂名/分子量/分子式/结构图管理
5. **标签系统**：多色标签与实验多对多关联
6. **导出功能**：ACS / JACS / Angewandte 三种期刊格式 + 自定义占位符模板

### 数据库表结构
projects → experiments (1:N)
experiments → reactants / catalysts / solvents (1:N, cascade delete)
experiments ↔ tags (M:N via experiment_tags)
独立表：templates、reagents

第二轮根本性优化（3项）：
