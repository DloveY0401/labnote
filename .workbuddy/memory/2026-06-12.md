### 修复: 确认并插入按钮不起作用 (第二次修复)
- **问题**: 上一轮模态转换后, 确认并插入按钮仍然无效
- **根本原因**:
  1. iframe 的 `sandbox="allow-scripts allow-same-origin"` 在 Electron 中阻止父窗口访问 `contentWindow.ketcher` API
  2. `handleCancel` 引用了已删除的 `isModal` 变量 (引用错误导致取消走 navigate 路径)
- **修复**:
  - `StructureDraw.tsx`: 移除 iframe sandbox 属性 (Ketcher 是可信本地资源)
  - `StructureDraw.tsx`: 新增 `saving` 状态, 按钮显示"保存中..."并提供禁用反馈
  - `StructureDraw.tsx`: 简化 `handleSave` 逻辑, 用 `onSave` 直接判断替代 `isModal` 变量
  - `StructureDraw.tsx`: 修复 `handleCancel` 中 `isModal` 引用为 `onCancel`

### 修复: 确认并插入需切换应用后才生效 (iframe 焦点事件路由)
- **症状**: 点击"确认并插入"无反应, 必须 alt+tab 切换应用再回来才执行保存
- **根因**: Electron 中 iframe 保持内部焦点时, Chromium 延迟父→子 contentWindow API 调用, 直到窗口重新获得焦点才释放
- **修复**:
  - `getKetcher()`: 访问 contentWindow 前调用 `iframe.contentWindow.focus()` 确保跨 frame 通信可用
  - `handleSave` / `refreshInfo`: 调用前先 `window.focus()` + 30ms 延迟确保焦点变更生效

### 优化: 确认并插入反应慢 (PubChem 阻塞)
- **症状**: 点击后等待十几秒才完成
- **根因**: `handleSave` 内 `await analyzeSmiles(smiles)` 同步等待 PubChem PUG REST API (SMILES→CID→IUPAC名称), 两次串行网络请求耗时10s+
- **修复**: 保存时仅用 `analyzeSmilesSync` 同步计算分子式/MW, 立即保存并关闭弹窗; PubChem 命名查询改为 fire-and-forget 后台异步, 结果仅更新右侧信息面板
