## 修复: 模板保存失败 (better-sqlite3 ABI 不匹配)
- **问题**: "No handler registered for 'templates:create'" — 数据库初始化失败导致所有 IPC handlers 跳过
- **根因**: `npmRebuild: false` 时 better-sqlite3 预编译二进制与 Electron 28 的 Node.js ABI 不匹配
- **修复**: `electron-builder.yml` 中 `npmRebuild: true`，electron-builder 自动下载匹配 Electron 版本的原生二进制
- 同时清理了 `scripts/strip-locales.js` 用于精简语言包
- **移除废弃依赖**: ketcher-standalone, ketcher-core, ketcher-react, @rdkit/rdkit (4个包)
  - 连带移除传递依赖: indigo-ketcher, @mui, paper, three, miew, draft-js, raphael 等
- **精简 Electron locales**: 通过 afterPack hook 仅保留 zh-CN + en-US (删除 53 个语言包)
- **效果**:
  - 安装后: 493MB → 256MB (↓48%)
  - 安装包: 117MB → 74MB (↓37%)
  - locales: 37MB → 876KB (↓98%)

## 桌面小组件透明化修复
- **问题**: widget 显示为白色不透明条，非透明毛玻璃效果
- **根因分析**:
  1. `index.css` 中 `html { background-color: #f8fafc }` 设置灰白背景 (Tailwind @layer base)
  2. `index.html` 中 `<body class="bg-gray-50">` 设置灰白背景
  3. `backdrop-filter: blur()` 在 Windows Electron 透明窗口上不可靠
  4. `resizable: false` 限制了窗口大小调整
- **修复**:
  - `electron/main.ts`: 添加 `backgroundColor: '#00000000'` 显式透明, 改为 `resizable: true`, 添加 resize 事件保持右对齐, 设置 `minWidth:240/maxWidth:500/minHeight:400`
  - `WidgetPage.tsx`: useEffect 中清除 html + body 背景 `transparent`, 移除 backdropFilter, 改用 `rgba(18,22,33,0.92)` 纯半透明背景 + 左侧细微边框

## 小组件显示黑色无内容修复
- **问题**: 小组件显示为黑色不透明条，没有内容
- **根因**: `BrowserWindow.loadFile(..., { hash: '/widget' })` 可能未正确设置 URL hash，导致 React Router HashRouter 无法匹配 `/widget` 路由，渲染空白
- **修复**: 生产环境改用 `loadURL(fileUrl + '#/widget')` 显式拼接 hash
- **打包**: 重建安装包 `LabNote-Setup-1.0.0.exe` (75MB)

## 小组件 require('electron') 崩溃修复
- **现象**: "闪一下内容"后变为空白黑色条
- **根因**: WidgetPage 中 `require('electron')` 在 contextIsolation:true + nodeIntegration:false 环境下报错 `ReferenceError: require is not defined`，React 卸载组件
- **修复**: 
  - `preload.ts`: 新增 `widget.onDataChanged(callback)` 方法，通过 contextBridge 暴露 ipcRenderer 事件
  - `types.ts`: 添加对应类型
  - `WidgetPage.tsx`: 用 `window.labnote.widget.onDataChanged()` 替换 `require('electron')`
- **打包**: 重建安装包 `LabNote-Setup-1.0.0.exe` (75MB)
