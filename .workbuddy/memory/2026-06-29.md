## 修复: 分子式不含 H (隐式氢计算)
- **问题**: SMILES 解析器只统计显式原子, 未计算隐式氢 (如苯 c1ccccc1 → C6 而非 C6H6)
- **根因**: 原 smilesToAtomCounts 仅逐字符计数元素符号, 不分析键级/化合价
- **修复**: 重写 smilesParser.ts
  - 新增完整 SMILES 语法解析器: 原子(方括号+有机子集)、键符(-=#:)、分支(()), 环闭合(数字)
  - 构建键图 (ParsedBond[]), 跟踪每个原子的键级总和
  - 隐式 H = 标准化合价 + 形式电荷 - 键级总和 - 显式氢
  - 分子式遵循 Hill 系统排序 (含C时C→H→字母序, 无C时纯字母序)
- 测试: 苯/乙酸/乙醇/吡啶/CO2/环己烷/苯甲酸 等常见分子公式正确

## 修复: 模块拖拽排序不可用
- **问题**: SectionHeader 拖拽手柄无法拖动模块排序
- **根因**: 使用 React state (`dragSourceIdx`) 存储拖拽源索引, 但 setState 异步, `drop` 事件触发时 state 可能未更新 → `if (dragSourceIdx === null) return` 提前退出
- **修复**: 改用浏览器原生 `dataTransfer.setData/getData` API 传递索引, 同步可靠, 不依赖 React 状态时序
  - `handleDragStart`: `e.dataTransfer.setData('text/plain', String(index))`
  - `handleDrop`: `parseInt(e.dataTransfer.getData('text/plain'), 10)`
  - 移除 `dragSourceIdx` state
