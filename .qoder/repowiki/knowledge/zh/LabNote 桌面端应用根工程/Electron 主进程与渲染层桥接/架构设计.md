模块由四个文件组成清晰的职责分层：
- `main.ts`：Electron 主进程入口，注册 `app`、`BrowserWindow`、`Menu`、`protocol.handle('labnote')` 自定义协议；维护 `mainWindow` 与 `widgetWindow`（无边框透明窗口，通过 PowerShell + user32.dll 嵌入 Windows 桌面）；集中注册所有 `ipcMain.handle` 处理器，按领域分组（projects/experiments/tags/templates/reagents/compound/tasks），并通过 `notifyWidget()` 向 widget 推送数据变更事件。
- `preload.ts`：通过 `contextBridge.exposeInMainWorld('labnote', api)` 暴露安全 API，定义 `LabNoteAPI` 接口作为渲染进程唯一访问面，所有方法内部仅调用 `ipcRenderer.invoke`，禁止直接访问 Node。
- `database.ts`：基于 `better-sqlite3` 的单例 DB 管理，`initDatabase(dataPath)` 在 WAL 模式 + foreign_keys=ON 下执行 DDL，并内置迁移逻辑（批量查询 `PRAGMA table_info` 后增量 `ALTER TABLE ADD COLUMN`，以及重建 `tags` 表以支持 `(name,type)` 复合唯一约束）；同时为 `module_templates` 预置表征/计算/安全/文献/物料清单等种子数据。
- `export.ts`：纯函数式文本格式化器，将实验结构化数据拼接成期刊风格的 Experimental Section，被 `experiments:export` IPC 调用。
依赖方向单向：`main.ts → {database.ts, export.ts}`，`preload.ts` 仅面向渲染进程，不反向引用 main。
