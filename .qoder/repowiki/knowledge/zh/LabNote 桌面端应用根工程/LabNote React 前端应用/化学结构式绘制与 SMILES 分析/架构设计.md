模块按职责分为三层：
- 页面层 `StructureDraw.tsx`：作为 Ketcher 编辑器的宿主，通过 `<iframe src="./ketcher/index.html">` 加载外部编辑器，使用 `waitForKetcher`/`getKetcher` 轮询 `contentWindow.ketcher` API 完成跨帧通信；负责初始 SMILES 注入、保存回调、信息面板刷新以及 Toast/ConfirmDialog 等 UI 组合。
- 渲染层 `MolCanvas.tsx`：纯 React + HTML5 Canvas 的自研分子画布，内部维护 `dragRef`/`firstAtomRef`/`ringRef` 等 ref 以桥接 React state 与 Canvas 事件循环；通过 `onMoleculeChange`/`onHistoryPush` 向上提交不可变 `Molecule` 变更。
- 数据与算法层 `utils/molEngine.ts` + `utils/smilesParser.ts`：前者定义 `Molecule`/`AtomData`/`BondData`/`ToolType`/`RingTemplate` 等核心类型并提供无副作用的原子/键增删改查、环模板插入、SMILES 生成 (`toSmiles`)、序列化/反序列化；后者实现完整的 SMILES 语法解析器（含分支、环闭合、方括号原子、芳香小写元素、键级 `- = # :`），并通过标准化合价表推导隐式氢，输出 Hill 分子式与分子量，同时通过本地 SQLite 缓存或 PubChem PUG REST 查询 IUPAC 名称。
依赖方向单向：页面 → 组件 → utils，组件不直接调用 SMILES 解析，仅消费 `molEngine` 导出的不可变模型。
